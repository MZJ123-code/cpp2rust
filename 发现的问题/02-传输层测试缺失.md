# 问题 02：TCP/IPC 传输层测试缺失

## 严重程度
🔴 严重

## 问题描述

当前性能测试只有 inproc（进程内）测试，缺少 TCP 和 IPC 传输层的性能测试，无法验证网络场景下的性能表现。

## 问题详情

### 当前测试覆盖

| 传输类型 | 延迟测试 | 吞吐测试 | 状态 |
|---------|---------|---------|------|
| inproc | ✅ inproc_lat | ✅ inproc_thr | 已完成 |
| TCP | ❌ | ❌ | 未实现 |
| IPC | ❌ | ❌ | 未实现 |

### 问题分析

1. **无法验证网络性能**
   - inproc 测试只验证进程内通信
   - TCP/IPC 涉及系统调用、内核缓冲区、网络栈
   - 实际部署场景主要是 TCP/IPC

2. **无法验证延迟**
   - 网络延迟是消息库的关键指标
   - 当前只有 inproc 延迟（8.414 μs）
   - TCP 延迟可能完全不同

3. **无法验证稳定性**
   - 网络场景可能有丢包、重传、拥塞
   - inproc 无法模拟这些场景

## 影响

- 无法验证实际部署场景的性能
- 无法对比 C++ 的 TCP/IPC 性能基线
- 用户无法评估是否适合生产使用

## 改进建议

### 方案 A：实现 TCP 性能测试

```rust
// perf/local_lat.rs
use std::net::TcpListener;
use std::thread;
use std::time::Instant;

fn main() {
    let args: Vec<String> = std::env::args().collect();
    let bind_addr = &args[1];
    let msg_size: usize = args[2].parse().unwrap();
    let msg_count: usize = args[3].parse().unwrap();

    let listener = TcpListener::bind(bind_addr).unwrap();
    let (mut stream, _) = listener.accept().unwrap();

    let mut buf = vec![0u8; msg_size];
    let start = Instant::now();

    for _ in 0..msg_count {
        stream.read_exact(&mut buf).unwrap();
    }

    let elapsed = start.elapsed();
    let latency = elapsed.as_micros() as f64 / msg_count as f64;
    println!("average latency: {:.3} [us]", latency);
}

// perf/remote_lat.rs
use std::net::TcpStream;
use std::time::Instant;

fn main() {
    let args: Vec<String> = std::env::args().collect();
    let connect_addr = &args[1];
    let msg_size: usize = args[2].parse().unwrap();
    let msg_count: usize = args[3].parse().unwrap();

    let mut stream = TcpStream::connect(connect_addr).unwrap();
    let buf = vec![0u8; msg_size];

    let start = Instant::now();

    for _ in 0..msg_count {
        stream.write_all(&buf).unwrap();
    }

    stream.flush().unwrap();
    println!("sent {} messages", msg_count);
}
```
