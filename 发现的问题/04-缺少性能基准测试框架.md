# 问题 04：缺少性能基准测试框架

## 严重程度
🟠 中等

## 问题描述

当前使用简单的 `cargo run --example` 进行性能测试，没有使用 Criterion 等专业基准测试框架，结果不易复现和比较。

## 问题详情

### 当前测试方式

```bash
# 手动运行示例
cargo run --example inproc_thr --release
cargo run --example inproc_lat --release
cargo run --example benchmark_radix_tree --release
```

### 问题分析

1. **结果不稳定**
   - 每次运行结果可能不同
   - 受系统负载影响
   - 无法进行统计分析

2. **无法比较历史结果**
   - 没有保存历史基准数据
   - 无法检测性能回归
   - 无法对比不同版本

3. **输出格式不标准**
   - 输出是简单的 println
   - 无法生成报告
   - 无法生成图表

4. **缺少统计信息**
   - 没有标准差
   - 没有置信区间
   - 没有百分位数

## 影响

- 无法可靠地比较性能
- 无法检测性能回归
- 无法生成专业的性能报告

## 改进建议

### 方案 A：使用 Criterion 框架

```toml
# Cargo.toml
[dev-dependencies]
criterion = { version = "0.5", features = ["html_reports"] }

[[bench]]
name = "inproc_throughput"
harness = false

[[bench]]
name = "inproc_latency"
harness = false

[[bench]]
name = "radix_tree"
harness = false
```

```rust
// benches/inproc_throughput.rs
use criterion::{criterion_group, criterion_main, Criterion, BenchmarkId};

fn bench_inproc_throughput(c: &mut Criterion) {
    let mut group = c.benchmark_group("inproc_throughput");

    for msg_size in [16, 32, 64, 128, 256, 1024] {
        group.bench_with_input(
            BenchmarkId::new("30b_vsm", msg_size),
            &msg_size,
            |b, &msg_size| {
                let ctx = ZContext::new();
                let push = ctx.socket(SocketType::PUSH).unwrap();
                let pull = ctx.socket(SocketType::PULL).unwrap();
                push.bind("inproc://bench").unwrap();
                pull.connect("inproc://bench").unwrap();

                let msg = vec![0u8; msg_size];

                b.iter(|| {
                    push.send(&msg, 0).unwrap();
                    pull.recv_bytes().unwrap();
                });
            },
        );
    }

    group.finish();
}

criterion_group!(benches, bench_inproc_throughput);
criterion_main!(benches);
```

### 方案 B：生成 HTML 报告

```bash
# 运行基准测试并生成报告
cargo bench --bench inproc_throughput

# 报告位于
target/criterion/inproc_throughput/report/index.html
```

### 方案 C：添加 CI/CD 集成

```yaml
# .github/workflows/benchmark.yml
name: Benchmark

on: [push, pull_request]

jobs:
  benchmark:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable

      - name: Run benchmarks
        run: cargo bench --bench all 2>&1 | tee output.txt

      - name: Store benchmark results
        uses: benchmark-action/github-action-benchmark@v1
        with:
          tool: cargo
          output-file-path: output.txt
          alert-threshold: '110%'
          fail-on-alert: true
```

### 方案 D：性能回归检测

```yaml
# .github/workflows/perf-regression.yml
name: Performance Regression

on: [pull_request]

jobs:
  check-regression:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: cargo bench --bench all > new_output.txt

      - name: Compare with main branch
        run: |
          git checkout main
          cargo bench --bench all > main_output.txt
          # 比较结果，检测性能回归
          python scripts/compare_benchmarks.py main_output.txt new_output.txt
```
