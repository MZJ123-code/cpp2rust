# 题目名称
将 libzmq 使用 Rust 重写

# 解题要求
1. 跨文件深度关联性上下文管理：在不破坏项目个模块调用关系的前提下，实现单点渐进式重构。
2. 编译/编译自愈合双向闭环：当遇到重构后库API更新带来的各种语法不兼容时，agent需要根据错误栈error stack 进行精准自愈，自动打补丁
3. 语义等价性保证：要求重构之后业务逻辑百分百不受破坏，且自生成覆盖全部主干路径的单元测试

# 交付件要求
1. libzmq_rust，可以按Rust构建成可执行文件
2. 测试用例也用rust，以及rust主流测试框架

# 验证用例
1. code\libzmq\tests，全部转成rust并测试通过
2. code\libzmq\perf，性能基线对比C++版本不小于95%
3. Rust unsafe比例小于10%
